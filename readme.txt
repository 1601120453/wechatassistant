Ryan Lin
2018年5月22日

需求：
1、部门内客服们没法使用统一的话术回答问题，于是想开发一个系统，能够存储所有的问题及回答，并且保存为PDF文件
这个pdf作为标准答案可在客服中间共享。

2、我有很多文档、项目资料，同时有很多学习资料和很多电脑。
我希望能有一个平台能够让我存放这些资料，并且很方便的进行检索

3、一些常用功能，比如刘昶给我发的pdf文件，是需要我转发到她的邮箱，
另外一些微信的功能，比如分享的网站、收藏、语句等，我希望能够进行留存，方便以后查看
这种日常性任务，希望可以在各个终端上进行调用

4、启动微信的机器作为宿主机，将宿主机部署在内网机器上，可以方便的访问其他机器，和远程操作其他机器

5、可以实现定时收发微信消息的功能，模拟在线

6、为了保障安全：
a.设置定时拦截器，每分钟只能接受10条命令；
b.能够在内存中保存一些历史消息，可以为以后的人工智能回答做准备；
c.超级管理员需要设定一个超级动态密码，或者超级算法，只有心算才能做到——以后可以扩展为使用数独或者其他谜题的答案

7、在windows或者linux，unix 操作系统中设置定时任务，当配置文件中的修复模式设置为“启动”的时候可以自动诊断服务健康状态，
如果服务宕了，则可自动从备份目录中恢复标准版

8、基于自我修复程序的健康运转，我可以实现在任意位置上传需要替换的文件，来将程序进行升级
